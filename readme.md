0. Створив клас FileNavigator. Клас призначений для зберігання списку файлів, які розташовані на конкретному шляху.

Наприклад: /path/to/file-> [files.txt, firstApp.java]



1. Створив клас FileData. Клас представляє конкретний файл і складається з: ім'я файлу, розміру в байтах, шлях до файлу.



2. Реалізував метод add у класі FileNavigator. Цей метод додає файл за вказаним шляхом. 
Якщо шлях вже існує, новий файл необхідно додати до списку, вже пов'язаному з відповідним шляхом. 
ВАЖЛИВО: Шлях – унікальне значення і не повинно повторюватися.



3. Реалізував метод find у класі FileNavigator. Метод повертає список файлів, пов'язаних з шляхом переданим як параметр.



4. Реалізував метод filterBySize у класі FileNavigator. Метод повертає список файлів, розмір (в байтах) яких не перевищує значення, передане як параметр.



5. Реалізував метод remove у класі FileNavigator. Метод видаляє шлях і пов'язані з ним файли, виходячи з значення шляху, переданого як параметр.



6. * Реалізував метод sortBySize у класі FileNavigator. Метод сортує всі наявні файли за розміром (за зростанням), потім повертає список відсортованих файлів.



7. ** Реалізував перевірку консистентності у методі add у класі FileNavigator. 
Не дозволяти додавати значення і повідомити про це в консолі, при спробі додати значення FileData значення шляху якого не відповідає шляху-ключу, що веде до списку файлів.

Наприклад:

Має бути помилка, оскільки шлях-ключ і шлях до файлу не співпадають

Шлях: /path/to/file

FileData: {name: ..., size: ..., path: /another/path/}
